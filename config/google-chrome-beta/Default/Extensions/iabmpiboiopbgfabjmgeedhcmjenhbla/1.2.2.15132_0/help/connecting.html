Establishing a new connection

To establish a new connection, enter an Address that is either:

  • A private network address if you are connecting over the same local network.
  • A public network address if you are connecting over the Internet.

Note: There are circumstances in which you must qualify the network address with a port number. If you see the message Connection refused (10061), try appending two colons and a number to the address, for example 192.168.2.187::80.

Change the Picture Quality if you want to control how the remote desktop is displayed (you can also tweak this mid-connection). Note that:

  • Automatic is the preferred choice, since VNC Viewer automatically adapts picture quality to network speed.
  • Let the other computer decide may enable connections to certain third-party VNC-compatible technology (such as VMWare) that would otherwise not be possible.

Click the Connect button. You may be asked to enter authentication credentials:

  • If you installed VNC on the remote computer and chose a Free license, this is the password you specified at install-time.
  • If you're trialing a Personal or an Enterprise license, this is the username and password you normally use to log on to your account on the remote computer.

You may also be asked to verify a signature uniquely identifying the remote computer. Note that only connections to VNC with a Personal or an Enterprise license are encrypted.

Note: Connecting may take a few moments the first time. Subsequently, it should be faster.

To find out how to control a connected computer, read these instructions. If you cannot connect, start with the online guide.
